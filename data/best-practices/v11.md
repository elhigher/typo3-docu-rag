# TYPO3 v11 Extension Development Best Practices

Synthesized from all 358 changelog entries (v11.0–11.5.x). Covers breaking changes, required migrations, and new APIs.

---

## 1. Extbase Controllers

**Return `ResponseInterface` from every action** — this is mandatory in v11. The old approach of returning nothing (void) is gone.

```php
use Psr\Http\Message\ResponseInterface;

public function listAction(): ResponseInterface
{
    $this->view->assign('items', $this->repo->findAll());
    return $this->htmlResponse();
}
```

Use `$this->htmlResponse()` for HTML responses. For JSON:
```php
return $this->jsonResponse(json_encode($data));
```

**Forward instead of `$this->forward()`** — use `ForwardResponse`:
```php
use TYPO3\CMS\Extbase\Http\ForwardResponse;

return new ForwardResponse('list');
// or with arguments and controller:
return (new ForwardResponse('show'))->withArguments(['id' => 1]);
```

**Switchable Controller Actions (SCA) are removed** — register separate plugins per action group instead. Use `PLUGIN_TYPE_CONTENT_ELEMENT` for content-element plugins.

**Plugin registration — use FQCN for controllers** (no vendor prefix in extension name):
```php
// ext_localconf.php
ExtensionUtility::configurePlugin(
    'MyExtension',                     // no vendor prefix
    'MyPlugin',
    [\Vendor\MyExtension\Controller\MyController::class => 'list,show'],
    [\Vendor\MyExtension\Controller\MyController::class => 'show']
);
```

**`BackendTemplateView` deprecated (v11.5)** — inject `ModuleTemplateFactory` instead:
```php
class MyController extends ActionController
{
    public function __construct(
        private readonly ModuleTemplateFactory $moduleTemplateFactory,
    ) {}

    public function indexAction(): ResponseInterface
    {
        $this->view->assign('var', 'value');
        $moduleTemplate = $this->moduleTemplateFactory->create($this->request);
        // add buttons, title, menus to $moduleTemplate ...
        $moduleTemplate->setContent($this->view->render());
        return $this->htmlResponse($moduleTemplate->renderContent());
    }
}
```

**`ControllerContext` deprecated (v11.5)** — access the request via `RenderingContext->getRequest()` in ViewHelpers; use `$this->request` in controllers.

**Extbase `ViewInterface` deprecated (v11.5)** — change `initializeView(ViewInterface $view)` signatures to `initializeView($view)` to avoid PHP contravariance errors in v12.

**Extbase validators with DI (v11.5.x)** — DI works again in v11, but requires:
```php
class MyValidator extends AbstractValidator
{
    public function __construct(private readonly MyService $service) {}

    public function setOptions(array $options): void
    {
        $this->initializeDefaultOptions($options);
    }
}
```
Plus in `Services.yaml`:
```yaml
MyVendor\MyExtension\Validation\Validator\MyValidator:
    public: true
    shared: false
```

## 2. PSR-14 Events (replace all hooks)

All `SC_OPTIONS` hooks are gone in v11. Use PSR-14 events instead.

**Register an event listener** in `Services.yaml`:
```yaml
MyVendor\MyExtension\EventListener\MyListener:
    tags:
        - name: event.listener
          identifier: 'my-extension/my-listener'
          event: TYPO3\CMS\Core\Mail\Event\AfterMailerSentMessageEvent
```

**Extbase Signals removed** — migrate to PSR-14 events:
```php
// Before
$this->signalSlotDispatcher->dispatch(static::class, 'mySignal', [$object]);

// After: dispatch your own event
$this->eventDispatcher->dispatch(new MyCustomEvent($object));
```

**Notable new PSR-14 events in v11:**
- `ModifyFileDumpEvent` — modify file dump response
- `ModifyClearCacheActionsEvent` — add/remove cache clear actions
- `AfterFileCommandProcessedEvent` — react to file operations
- `ModifyRecordListTableActionsEvent` / `ModifyRecordListHeaderColumnsEvent` / `ModifyRecordListRecordActionsEvent` — customize record list
- `ModifyResolvedFrontendGroupsEvent` — attach frontend user groups dynamically (v11.5)
- `BeforeSectionMarkupGeneratedEvent` / `AfterSectionMarkupGeneratedEvent` — use `getPageLayoutContext()` and `getRecords()` (old `getPageLayoutView()` deprecated)

## 3. Dependency Injection

**Constructor injection — mandatory** for all new code. `@Inject` on non-public properties is removed:
```php
// WRONG in v11+
/** @Inject */
protected MyService $service;

// CORRECT
public function __construct(private readonly MyService $service) {}
```

**`getInstance()` static methods deprecated (v11.5)** — inject via DI or use `GeneralUtility::makeInstance()`:
```php
// Deprecated:
ExtractorRegistry::getInstance()->...

// Use DI:
public function __construct(private readonly ExtractorRegistry $extractorRegistry) {}
```

**`ModuleTemplate->getIconFactory()` / `getPageRenderer()` deprecated (v11.5)** — inject `IconFactory` and `PageRenderer` directly:
```php
public function __construct(
    private readonly ModuleTemplateFactory $moduleTemplateFactory,
    private readonly IconFactory $iconFactory,
    private readonly PageRenderer $pageRenderer,
) {}
```

## 4. TCA

**TCA type `category`** — replaces `CategoryRegistry::getInstance()->makeCategorizable()`:
```php
// Old (removed in v12, deprecated in v11)
\TYPO3\CMS\Core\Category\CategoryRegistry::getInstance()->makeCategorizable(
    'my_extension', 'tx_myextension_domain_model_item'
);

// New — in TCA (e.g. TCA/Overrides/tx_myextension_domain_model_item.php)
'categories' => [
    'config' => [
        'type' => 'category',
    ],
],
```

**TCA `type=group` — `internal_type=db` is now optional (v11.5)** — `db` is the default; remove this option from TCA `group` fields.

**MM tables auto-created from TCA** — remove manual `CREATE TABLE` for MM tables from `ext_tables.sql`. TYPO3 generates the table from TCA.

## 5. Icons

**Register icons via `Configuration/Icons.php`** (resolved at cache-warmup time, not ext_localconf.php):

```php
// Configuration/Icons.php
return [
    'my-icon-identifier' => [
        'provider' => \TYPO3\CMS\Core\Imaging\IconProvider\SvgIconProvider::class,
        'source' => 'EXT:my_extension/Resources/Public/Icons/my-icon.svg',
    ],
];
```

This is faster than `IconRegistry` calls in `ext_localconf.php`.

## 6. Sessions

**`UserSession` / `UserSessionManager` API** — replaces direct array access on session data:
```php
// Inject UserSessionManager for frontend sessions
/** @var \TYPO3\CMS\Core\Session\UserSessionManager */
$sessionManager = UserSessionManager::create('FE');

$session = $sessionManager->createFromRequestOrAnonymous($request, 'fe_typo_user');
$data = $session->get('myKey');
```

## 7. Logging

**Inject `LoggerInterface` in constructor** — no more `LoggerAwareTrait` required:
```php
use Psr\Log\LoggerInterface;

public function __construct(private readonly LoggerInterface $logger) {}
```

**`#[Channel('name')]` PHP attribute** — group log entries into named channels:
```php
use TYPO3\CMS\Core\Log\Channel;

#[Channel('my-extension')]
class MyService
{
    public function __construct(private readonly LoggerInterface $logger) {}
}
```

## 8. ext_localconf.php / ext_tables.php Scoping

**`ext_localconf.php` is scoped to the global namespace** — you may use `use` statements at the top:
```php
<?php
use TYPO3\CMS\Extbase\Utility\ExtensionUtility;

ExtensionUtility::configurePlugin(/* ... */);
```

**Must NOT be wrapped in a custom namespace** — this breaks v11+:
```php
// WRONG
namespace MyVendor\MyExtension;
call_user_func(function () {
    // ...
});
```

**`$_EXTKEY` is no longer set** in `ext_localconf.php` / `ext_tables.php` — use the extension key string literal directly:
```php
// Old (broken)
\TYPO3\CMS\Core\Utility\ExtensionManagementUtility::addPlugin(..., $_EXTKEY);

// Correct
\TYPO3\CMS\Core\Utility\ExtensionManagementUtility::addPlugin(..., 'my_extension');
```

## 9. Content Objects / TypoScript

**TEMPLATE cObject removed** — use `FLUIDTEMPLATE`:
```typoscript
# Old (removed)
page.10 = TEMPLATE
page.10.template.file = ...

# New
page.10 = FLUIDTEMPLATE
page.10.templateName = Default
page.10.templateRootPaths.10 = EXT:my_extension/Resources/Private/Templates/
```

**`f:link.external` / `f:uri.external` default scheme is now `https`** — explicit `scheme="http"` needed for non-HTTPS links.

**`f:uri.email` deprecated (v11.5)** — use `<f:link.email email="{email}">` (returns full `<a>` tag with spam protection) or render the email directly in the template.

**`parseFunc.sword` deprecated (v11.5)** — implement search-word highlighting client-side.

**`page.includeCSS/includeCSSLibs.import = 1` deprecated (v11.5)** — use the `<link>` tag syntax instead.

**`<f:transform.html>` view helper (v11.5)** — resolves `t3://` URIs in HTML without TypoScript:
```html
<f:transform.html selector="a.href,div.data-uri">
    <a href="t3://page?uid=1">Visit page</a>
</f:transform.html>
```

## 10. userFunc / PSR-7 Request Access

**PSR-7 request is passed as the 3rd argument to userFunc callbacks**:
```php
public function myUserFunc(string $content, array $conf, ServerRequestInterface $request): string
{
    // access $request directly
}
```

## 11. Backend Modules & URLs

**Backend URL structure is fully path-based** — all backend routes now use path-based URLs. Always use `UriBuilder` to generate backend links; hardcoded URL strings with `?module=...` will break.

**`navigationFrameModule` deprecated (v11.5)** — migrate navigation iFrames to Web Components using `navigationComponentId`.

**Backend module registration** — use `ext_tables.php` with `ExtensionManagementUtility::addModule()` and reference a PHP controller class, or use new module registration via PHP attributes in v12+.

## 12. PHP Utility Replacements (v11.5 deprecations)

| Old | New |
|-----|-----|
| `GeneralUtility::isFirstPartOfStr($str, $prefix)` | `str_starts_with($str, $prefix)` |
| `StringUtility::beginsWith($str, $prefix)` | `str_starts_with($str, $prefix)` |
| `StringUtility::endsWith($str, $suffix)` | `str_ends_with($str, $suffix)` |
| `GeneralUtility::isAbsPath($path)` | `PathUtility::isAbsolutePath($path)` |
| `FlexFormTools->getArrayValueByPath($path, $arr)` | `ArrayUtility::getValueByPath($arr, $path)` |
| `FlexFormTools->setArrayValueByPath($path, $arr, $val)` | `ArrayUtility::setValueByPath($arr, $path, $val)` |
| `GeneralUtility::minifyJavaScript()` | Use `PageRenderer` / `ResourceCompressor` API |

## 13. Upgrade Wizards

**`class.ext_update.php` removed** — all upgrade logic must use the Upgrade Wizard API:
```php
class MyUpgradeWizard implements UpgradeWizardInterface
{
    public function getIdentifier(): string { return 'myExtension_myUpgradeWizard'; }
    public function getTitle(): string { return 'My Upgrade Wizard'; }
    public function getDescription(): string { return '...'; }
    public function executeUpdate(): bool { /* do migration */ return true; }
    public function updateNecessary(): bool { /* check if needed */ return false; }
    public function getPrerequisites(): array { return [DatabaseUpdatedPrerequisite::class]; }
}
```

Register in `Services.yaml`:
```yaml
MyVendor\MyExtension\Updates\MyUpgradeWizard:
    tags:
        - name:升级.wizard
```
(Actually: use `name: 'typo3.upgradewizard'` or register via `$GLOBALS['TYPO3_CONF_VARS']['SC_OPTIONS']['ext/install']['update']`.)

## 14. Composer Mode

**`PackageStates.php` removed** in Composer mode — TYPO3 no longer manages this file when running in Composer mode. All packages declared in `composer.json` are always active.

**`ext_emconf.php` ignored** in Composer mode — package metadata comes from `composer.json` only.

**`extension:setup` CLI command** — run after `composer require` to set up a newly installed extension:
```bash
vendor/bin/typo3 extension:setup
```

**All Composer-managed extensions are always active** — the "activate/deactivate" Extension Manager toggle has no effect in Composer mode.

## 15. Security Notes

**Site resolution by query parameters restricted (v11.5.30 security patch)** — pages can no longer be resolved via `?id=123&L=1` cross-site. If your setup requires this, enable the feature flag (not recommended):
```php
$GLOBALS['TYPO3_CONF_VARS']['SYS']['features']['security.frontend.allowInsecureSiteResolutionByQueryParameters'] = true;
```

**FAL enforces absolute paths within project root** (v11.5.x) — file storages with absolute paths must be within `TYPO3_PATH_ROOT` or listed in `lockRootPath`:
```php
$GLOBALS['TYPO3_CONF_VARS']['BE']['lockRootPath'] = [
    '/var/extra-storage1/',
    '/var/extra-storage2/',
];
```

**Public assets must be in `Resources/Public/`** (v11.5.x) — assets outside this folder will trigger a deprecation. Move all web-accessible files into `Resources/Public/`.

**GFX strip color profile parameters** (v11.5.x) — use array syntax:
```php
$GLOBALS['TYPO3_CONF_VARS']['GFX']['processor_stripColorProfileParameters'] = ['+profile', '*'];
```

**Export restricted to admins** (v11.5.11 security patch) — non-admin users need explicit TSconfig:
```
options.impexp.enableExportForNonAdminUser = 1
```

## 16. Fluid ViewHelpers

**Do not extend core ViewHelpers** — all core ViewHelpers will be declared `final` in v12. Use Fluid abstract base classes for custom ViewHelpers instead.

**`<f:transform.html>` available** — resolves `t3://` links in arbitrary HTML, usable in both frontend and backend without TypoScript setup.

---

## 17. Frontend User Authentication

**`UserSessionManager` / `UserSession` API** — replaces the old session array approach. Use typed session objects returned by `UserSessionManager::create('FE')`.

---

## 18. felogin

**felogin is Extbase-based** — the old `pi_base` implementation is gone. Override templates in `EXT:felogin/Resources/Private/Templates/` path.

**TypoScript setting renamed** (v11.5.x) — `showForgotPasswordLink` → `showForgotPassword`:
```typoscript
plugin.tx_felogin_login.settings.showForgotPassword = 1
```

---

## 19. FormEngine / JavaScript (Backend)

**`fieldChangeFunc` must use `OnFieldChangeInterface`** (v11.5) — scalar string values (inline JS) are deprecated:
```php
// Old (deprecated)
$this->data['parameterArray']['fieldChangeFunc']['example'] = "alert('demo');";

// New — implement OnFieldChangeInterface
$this->data['parameterArray']['fieldChangeFunc']['example'] = new MyOnFieldChange();
```

**`AbstractControl::setOnClick` deprecated** — use `data-dispatch-action` / `data-dispatch-args` HTML attributes with `ActionDispatcher`, or `PreviewUriBuilder` for page preview.

**RequireJS callbacks as inline JS deprecated** — use `JavaScriptModuleInstruction`:
```php
use TYPO3\CMS\Core\Page\JavaScriptModuleInstruction;

$resultArray['requireJsModules'][] = JavaScriptModuleInstruction::forRequireJS(
    'TYPO3/CMS/Backend/FormEngine/Element/InputDateTimeElement'
)->instance($fieldId);
```

---

## 20. Default URI Scheme

**Configurable default scheme** (v11.5.x) — change fallback protocol for generated links:
```php
$GLOBALS['TYPO3_CONF_VARS']['SYS']['defaultScheme'] = 'https';
```

## Quick Reference

|------|-----|-----|
| Extbase action return | `void` | `ResponseInterface` |
| Extbase forward | `$this->forward('action')` | `return new ForwardResponse('action')` |
| Extbase SCA | `switchableControllerActions` | Separate plugins per action group |
| Plugin controller keys | Short class name | FQCN |
| ext_update.php | `class.ext_update.php` | `UpgradeWizardInterface` |
| Category field | `makeCategorizable()` | TCA type `category` |
| Icon registration | `ext_localconf.php` call | `Configuration/Icons.php` |
| MM table SQL | `CREATE TABLE` in ext_tables.sql | Auto-created from TCA |
| Logger | `LoggerAwareTrait` | Constructor-inject `LoggerInterface` |
| Session | Array-based | `UserSession` / `UserSessionManager` |
| SC_OPTIONS hooks | Hook arrays | PSR-14 events |
| Extbase signals | `signalSlotDispatcher` | PSR-14 events |
| `@Inject` | Non-public properties | Constructor injection |
| TEMPLATE cObject | `= TEMPLATE` | `= FLUIDTEMPLATE` |
| Fluid widgets | `<f:widget.*>` | Custom ViewHelpers + Pagination API |
| Extbase BackendTemplateView | `$this->view->getModuleTemplate()` | Inject `ModuleTemplateFactory` |
| `getInstance()` statics | `Registry::getInstance()` | DI / `GeneralUtility::makeInstance()` |
| `str_starts_with` | `GeneralUtility::isFirstPartOfStr()` | `str_starts_with()` |
| `str_ends_with` | `StringUtility::endsWith()` | `str_ends_with()` |

| Composer mode | `PackageStates.php` | `extension:setup` CLI |
| felogin TypoScript | `showForgotPasswordLink` | `showForgotPassword` |
