# TYPO3 12.x Extension Development Best Practices

Derived from a thorough read of all 526 TYPO3 v12.x changelog entries (12.0ŌĆō12.4.x).

---

## Plugin Registration ŌĆö Drop AbstractPlugin

`AbstractPlugin` (pibase, `tslib_pibase`) was marked `@internal` in v12.0 and deprecated in v12.4. It is removed in v13.0. Do not create new plugins with it.

Use Extbase with a dedicated `CType` per plugin:

```php
// ext_localconf.php
\TYPO3\CMS\Extbase\Utility\ExtensionUtility::configurePlugin(
    'MyExt',
    'MyPlugin',
    [MyController::class => 'index,show'],
);

// Configuration/TCA/Overrides/tt_content.php
\TYPO3\CMS\Extbase\Utility\ExtensionUtility::registerPlugin(
    'MyExt',
    'MyPlugin',
    'LLL:EXT:my_ext/Resources/Private/Language/locallang.xlf:plugin.title',
    'my-plugin-icon',
    'plugins',
);
```

## PSR-14 Events ŌĆö All Hooks Are Gone

`$GLOBALS['TYPO3_CONF_VARS']['SC_OPTIONS']` hooks were removed in v12.0. Register PSR-14 event listeners in `Services.yaml` or (from v13) with `#[AsEventListener]`:

```yaml
# Services.yaml (v12)
MyVendor\MyExt\EventListener\MyListener:
  tags:
    - name: event.listener
      identifier: 'my-ext/my-listener'
      event: TYPO3\CMS\Core\Mail\Event\AfterMailerSentMessageEvent
```

Search the v12.0 Breaking changelog for the specific event that replaced the hook you were using ŌĆö almost every hook has a 1-to-1 PSR-14 replacement.

## New TCA Field Types

### `type = 'file'` (v12.0) ŌĆö replaces inline+FAL pattern

```php
'images' => [
    'label' => 'Images',
    'config' => [
        'type' => 'file',
        'allowed' => 'common-image-types',
        'maxitems' => 5,
    ],
],
```

No more `type=inline` with `foreign_table=sys_file_reference` and the manual FAL `overrideChildTca`. The `file` type handles everything.

### `type = 'json'` (v12.3) ŌĆö native JSON storage

```php
'meta_data' => [
    'label' => 'Meta data',
    'config' => ['type' => 'json'],
],
```

### `type = 'password'` with policy validation

```php
'my_password' => [
    'config' => [
        'type' => 'password',
        'passwordPolicy' => 'default',
    ],
],
```

Password policies are configured in `$GLOBALS['TYPO3_CONF_VARS']['SYS']['passwordPolicies']`.

## TCA ŌĆö Associative Items Array (v12.3)

`items` arrays in `select`, `radio`, and `check` fields accept associative syntax. Use it for clarity:

```php
'items' => [
    [
        'label' => 'LLL:EXT:my_ext/locallang.xlf:option.first',
        'value' => 'first',
        'icon'  => 'myext-icon-first',
        'group' => 'myGroup',
        'description' => 'LLL:EXT:my_ext/locallang.xlf:option.first.description',
    ],
],
```

```php
// old
'config' => ['type' => 'input', 'eval' => 'null'],

// new
'config' => ['type' => 'input', 'nullable' => true],
```

## TCA ŌĆö Remove `MM_insert_fields` and `MM_match_fields`

The `MM_insert_fields` and `MM_match_fields` options were removed. Use dedicated MM tables without extra insert fields ŌĆö store relation metadata in separate columns instead.

## TCA ŌĆö FlexForm: Drop `<TCEforms>` Wrapper

Remove the legacy `<TCEforms>` wrapper element from FlexForm XML. Write the field configuration directly:

```xml
<!-- old -->
<settings.myField>
  <TCEforms>
    <label>My Field</label>
    <config>...</config>
  </TCEforms>
</settings.myField>

<!-- new -->
<settings.myField>
  <label>My Field</label>
  <config>...</config>
</settings.myField>
```

## TCA ŌĆö `PageDoktypeRegistry` Instead of `$GLOBALS['PAGES_TYPES']`

```php
// old
$GLOBALS['PAGES_TYPES'][254] = ['type' => 'sys_folder', 'allowedTables' => '*'];

// new ŌĆö Services.yaml
MyVendor\MyExt\PageType\MyPageTypeProvider:
  tags:
    - name: pages.doktype
      doktype: 254
```

Or use `PageDoktypeRegistry` directly via dependency injection.

## JavaScript ŌĆö Native ES Modules, No RequireJS

RequireJS/AMD is deprecated in v12 and removed in v13. Use native ES modules with `@typo3/` package imports:

```js
// old
define(['TYPO3/CMS/MyExt/MyModule'], function(MyModule) { ... });

// new
import MyModule from '@typo3/my-ext/my-module.js';
```

jQuery is no longer available as `window.jQuery` or `window.$`. Import it as a module if needed:

```js
import $ from '@typo3/core/contrib/jquery/jquery.esm.js';
```

For FormEngine result arrays, use `javaScriptModules` (not the legacy `requireJsModules`):

```php
$result['javaScriptModules'][] = JavaScriptModuleInstruction::create(
    '@my-vendor/my-ext/my-field-control.js'
)->invoke('initialize');
```

## Flash Messages ŌĆö `ContextualFeedbackSeverity` Enum

Replace integer constants with the enum:

```php
// old
use TYPO3\CMS\Core\Messaging\FlashMessage;
new FlashMessage('text', 'title', FlashMessage::OK);
new FlashMessage('text', 'title', FlashMessage::WARNING);
new FlashMessage('text', 'title', FlashMessage::ERROR);

// new
use TYPO3\CMS\Core\Type\ContextualFeedbackSeverity;
new FlashMessage('text', 'title', ContextualFeedbackSeverity::OK);
new FlashMessage('text', 'title', ContextualFeedbackSeverity::WARNING);
new FlashMessage('text', 'title', ContextualFeedbackSeverity::ERROR);
```

## Configuration File Locations (Composer Mode)

| Old path | New path |
|---|---|
| `typo3conf/LocalConfiguration.php` | `config/system/settings.php` |
| `typo3conf/AdditionalConfiguration.php` | `config/system/additional.php` |
| `typo3conf/PackageStates.php` | Managed by Composer ŌĆö do not edit |

Extension assets in Composer mode land in `public/_assets/<hash>/` (not `typo3/` subdirectory symlinks). Do not hardcode asset paths.

## Backend Module Registration

Do **not** use `$GLOBALS['TBE_MODULES']`, `TBE_MODULES_EXT`, or `ExtensionManagementUtility::insertModuleFunction()`. Register modules in `Configuration/Backend/Modules.php`:

```php
// Configuration/Backend/Modules.php
return [
    'myext_mymodule' => [
        'parent' => 'web',
        'position' => ['after' => 'web_layout'],
        'access' => 'user',
        'path' => '/module/myext/mymodule',
        'iconIdentifier' => 'myext-module-icon',
        'labels' => 'LLL:EXT:my_ext/Resources/Private/Language/locallang_mod.xlf',
        'routes' => [
            '_default' => [
                'target' => MyController::class . '::handleRequest',
            ],
        ],
    ],
];
```

## Extension Icon Location

Place the extension icon at `Resources/Public/Icons/Extension.svg` (or `.png`/`.gif`). Root-level `ext_icon.*` is deprecated in v12.4.

## Service Registration via DI Tags

Many subsystems that previously required `ext_localconf.php` or `ext_tables.php` calls now use DI service tags:

```yaml
# Services.yaml

# System status report
MyVendor\MyExt\Report\MyReport:
  tags:
    - name: report.status

# Link validator linktype
MyVendor\MyExt\LinkValidator\MyLinkType:
  tags:
    - name: linkvalidator.linktype
      identifier: 'myExt'

# Element browser
MyVendor\MyExt\Browser\MyBrowser:
  tags:
    - name: recordlist.elementbrowser
      identifier: 'myExt'
      before: 'db'
```

## Site Settings ŌĆö `settings.yaml`

From v12.1, site settings are separated from `config.yaml` into `config/sites/<siteidentifier>/settings.yaml`. Access them at runtime via:

```php
$value = $site->getSettings()->get('my_vendor.my_ext.my_setting');
```

Define settings with types and defaults in `settings.yaml`:

```yaml
# config/sites/mysite/settings.yaml
my_vendor:
  my_ext:
    my_setting: 'default value'
```

## Content Security Policy (v12.3)

Register CSP policies per extension in `Configuration/ContentSecurityPolicies.php`:

```php
// Configuration/ContentSecurityPolicies.php
use TYPO3\CMS\Core\Security\ContentSecurityPolicy\Directive;
use TYPO3\CMS\Core\Security\ContentSecurityPolicy\Mutation;
use TYPO3\CMS\Core\Security\ContentSecurityPolicy\MutationCollection;
use TYPO3\CMS\Core\Security\ContentSecurityPolicy\MutationMode;
use TYPO3\CMS\Core\Security\ContentSecurityPolicy\PolicyProvider;
use TYPO3\CMS\Core\Security\ContentSecurityPolicy\SourceKeyword;
use TYPO3\CMS\Core\Security\ContentSecurityPolicy\UriValue;
use TYPO3\CMS\Core\Type\Map;

return new class implements PolicyProvider {
    public function getPolicy(Map $mutations): MutationCollection
    {
        return new MutationCollection(
            new Mutation(
                MutationMode::Extend,
                Directive::ImgSrc,
                new UriValue('https://cdn.example.com'),
            ),
        );
    }
};
```

Do not use inline `<script>` or `<style>` without nonces. Do not disable CSP globally.

## Extbase Improvements

### Enum action arguments (v12.2)

Controller action methods can type-hint enum parameters directly ŌĆö Extbase resolves and validates automatically:

```php
public function showAction(StatusEnum $status): ResponseInterface { ... }
```

### Union types in domain model properties

Extbase now maps union-typed properties correctly. You no longer need single-type wrapper objects for optional relations.

### Language aspect on query settings

```php
$querySettings->setLanguageAspect($languageAspect);
```

Replaces deprecated individual `setRespectSysLanguage()` / `setLanguageUid()` calls.

## TypoScript

### New parser (v12.0)

The TypoScript parser was fully rewritten. Syntax errors now emit proper error messages. Ensure your TypoScript is syntactically correct ŌĆö previously silently-ignored errors will surface.

### New `if` conditions

```typoscript
[request.getQueryParams()['tx_myext'] contains 'foo']
[request.getQueryParams()['tx_myext'] startsWith 'bar']
[request.getQueryParams()['tx_myext'] endsWith 'baz']
```

### Deprecated condition variables ŌĆö use context API

```typoscript
# deprecated
[loginUser('*')]
[usergroup('42')]

# new
[frontend.user.isLoggedIn]
[frontend.user.isInGroup(42)]
[backend.user.isLoggedIn]
[backend.user.isAdmin]
```

## Database ŌĆö ExpressionBuilder

```php
// deprecated
$queryBuilder->expr()->andX($expr1, $expr2);
$queryBuilder->expr()->orX($expr1, $expr2);

// new
$queryBuilder->expr()->and($expr1, $expr2);
$queryBuilder->expr()->or($expr1, $expr2);
```

## Request Access ŌĆö Drop `GeneralUtility::_GET()` / `_POST()`

```php
// deprecated
$value = \TYPO3\CMS\Core\Utility\GeneralUtility::_GET('my_param');

// new ŌĆö inject ServerRequestInterface into your controller/middleware
$value = $request->getQueryParams()['my_param'] ?? null;
$value = $request->getParsedBody()['my_param'] ?? null;
```

---

## Symfony Console ŌĆö Return `int` from `execute()`

Symfony 6 (shipped with TYPO3 v12) requires `execute()` to return an integer exit code:

```php
protected function execute(InputInterface $input, OutputInterface $output): int
{
    // ...
    return Command::SUCCESS; // 0
}
```

Commands that return `void` or `null` will trigger a deprecation.

## Country API

TYPO3 v12 ships a `CountryProvider` service with ISO 3166-1 country data. Inject and use it instead of maintaining your own country list:

```php
use TYPO3\CMS\Core\Country\CountryProvider;

final class MyService
{
    public function __construct(private readonly CountryProvider $countryProvider) {}

    public function getGermany(): Country
    {
        return $this->countryProvider->getByIsoCode('DE');
    }
}
```

## Security

### FAL MIME type enforcement

Since v12.2 (hardened in 12.4.x), uploaded files must have a MIME type consistent with their extension. Accepting uploads with mismatched types will be rejected. Test your upload forms.

### CSP sandbox on fileadmin

File previews and embedded files from fileadmin are served in a CSP `sandbox` iframe. Do not assume embedded files can run scripts.

### Extension Manager protected by sudo mode

Actions in the Extension Manager require sudo mode re-authentication. Extensions that drove EM actions programmatically need to account for this.
