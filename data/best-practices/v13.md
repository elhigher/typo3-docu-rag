# TYPO3 13.4 Extension Development Best Practices

Derived from a thorough read of all 356 TYPO3 13.x changelog entries (13.0–13.4.x).

---

## Plugin Registration

**Do not use the legacy `list`/`list_type` approach.** Since v12.4 the recommended way is a dedicated `CType` per plugin. The old "General Plugin" content element (`list`) and `list_type` sub types are deprecated in 13.4 and will be removed in v14.

```php
// ext_localconf.php
\TYPO3\CMS\Extbase\Utility\ExtensionUtility::configurePlugin(
    'MyExt',
    'MyPlugin',
    [MyController::class => 'index,show'],
);

// Configuration/TCA/Overrides/tt_content.php
\TYPO3\CMS\Extbase\Utility\ExtensionUtility::registerPlugin(
    'MyExt',
    'MyPlugin',
    'LLL:EXT:my_ext/Resources/Private/Language/locallang.xlf:plugin.title',
    'my-plugin-icon',
    'plugins',
);
```

## PSR-14 Events Instead of Hooks

All hooks are gone. Use PSR-14 events registered via the `#[AsEventListener]` PHP attribute (TYPO3 13+):

```php
use TYPO3\CMS\Core\Attribute\AsEventListener;

#[AsEventListener(identifier: 'my-ext/modify-something')]
final class MyEventListener
{
    public function __invoke(SomeEvent $event): void
    {
        // ...
    }
}
```

No manual `Services.yaml` entry is needed when using the attribute. The class must be in a DI-autoconfigured path (default: `Classes/`).

---

## Dependency Injection

Use constructor injection. Tag services with `#[Autoconfigure]` or rely on `Services.yaml` autoconfiguration. Do not use `GeneralUtility::makeInstance()` for anything that can be injected.

```php
use TYPO3\CMS\Core\Attribute\AsController; // for Extbase controllers

#[AsController]
final class MyController extends ActionController {}
```

## View / Fluid

### Use ViewFactoryInterface — not StandaloneView

`StandaloneView` is deprecated. Inject `\TYPO3\CMS\Core\View\ViewFactoryInterface` instead:

```php
use TYPO3\CMS\Core\View\ViewFactoryInterface;
use TYPO3\CMS\Core\View\ViewFactoryData;

final class MyService
{
    public function __construct(private readonly ViewFactoryInterface $viewFactory) {}

    public function render(ServerRequestInterface $request): string
    {
        $view = $this->viewFactory->create(new ViewFactoryData(
            templateRootPaths: ['EXT:my_ext/Resources/Private/Templates/'],
            partialRootPaths:  ['EXT:my_ext/Resources/Private/Partials/'],
            layoutRootPaths:   ['EXT:my_ext/Resources/Private/Layouts/'],
            request: $request,
        ));
        $view->assign('foo', 'bar');
        return $view->render('MyTemplate');
    }
}
```

### ModuleTemplate::renderResponse() requires explicit template filename

Since 13.4 the `$templateFileName` argument is mandatory:

```php
return $this->moduleTemplate->renderResponse('List'); // not empty string
```

### ViewHelper: drop renderStatic()

`renderStatic()` is removed. Implement `render()` only:

```php
final class MyViewHelper extends AbstractViewHelper
{
    public function render(): string
    {
        // use $this->arguments, $this->renderChildren(), etc.
    }
}
```

### Fluid 4 boolean attributes

```html
<my:vh async="{true}" />   <!-- renders: async="async" -->
<my:vh async="{false}" />  <!-- attribute omitted entirely -->
```

## TypoScript

### Use `@import` — not `<INCLUDE_TYPOSCRIPT:`

`INCLUDE_TYPOSCRIPT` is deprecated in 13.4 and will be removed in v14:

```typoscript
# old (deprecated)
<INCLUDE_TYPOSCRIPT: source="FILE:EXT:my_ext/Configuration/TypoScript/setup.typoscript">

# new
@import 'EXT:my_ext/Configuration/TypoScript/setup.typoscript'
```

### PAGEVIEW content object

Use `PAGEVIEW` for rendering page-level content in Fluid instead of legacy `FLUIDTEMPLATE` at the page level:

```typoscript
page = PAGE
page.10 = PAGEVIEW
page.10.paths.10 = EXT:my_ext/Resources/Private/Templates/
```

### Frontend cache tagging

From 13.3, database records used in frontend rendering are tagged automatically. Do not manually add cache tags for standard record relations. Use the `CacheDataCollector` API (`frontend.cache.collector` request attribute) if you need custom cache tags:

```php
/** @var \TYPO3\CMS\Frontend\Cache\CacheDataCollector $collector */
$collector = $request->getAttribute('frontend.cache.collector');
$collector->addCacheTags(new \TYPO3\CMS\Core\Cache\CacheTag('my_tag', 3600));
```

Do **not** call deprecated `$GLOBALS['TSFE']->addCacheTags()`.

---

## Avoid $GLOBALS['TSFE'] / TypoScriptFrontendController

`TypoScriptFrontendController` and `$GLOBALS['TSFE']` are deprecated in 13.4 and will be removed in v14. Use PSR-7 request attributes instead:

| Old (`$GLOBALS['TSFE']`) | New (request attribute) |
|---|---|
| `$GLOBALS['TSFE']->page` | `$request->getAttribute('frontend.page.information')->getPage()` |
| `$GLOBALS['TSFE']->id` | `$request->getAttribute('frontend.page.information')->getId()` |
| `$GLOBALS['TSFE']->sys_language_uid` | `$request->getAttribute('language')->getLanguageId()` |
| `$GLOBALS['TSFE']->addCacheTags(...)` | `$request->getAttribute('frontend.cache.collector')->addCacheTags(...)` |

## TCA

### System columns are auto-created

Do **not** manually define `starttime`, `endtime`, `hidden`, `fe_group`, `sys_language_uid`, `l10n_parent`, etc. columns in `TCA['columns']` if you simply need the standard behaviour. TYPO3 creates them automatically based on `ctrl` settings:

```php
// Only define ctrl entries — columns appear automatically
'ctrl' => [
    'enablecolumns' => ['disabled' => 'hidden', 'starttime' => 'starttime', 'endtime' => 'endtime'],
    'languageField' => 'sys_language_uid',
    'transOrigPointerField' => 'l10n_parent',
],
```

### System fields in showitem

System fields (language, hidden, etc.) are now added automatically to `showitem` based on `ctrl`. You no longer need to add them manually in content type definitions.

### No TCA sub types for plugins

Do not use `subtype_value_field` / `subtypes_addlist` for plugin registration. Use dedicated `CType` entries (see Plugin Registration above).

### Opacity in color fields

```php
'my_color' => [
    'config' => ['type' => 'color', 'opacity' => true],
],
```

## PHP Enums — Drop Enumeration Class

Replace any `\TYPO3\CMS\Core\Type\Enumeration` subclasses with native PHP 8.1 backed enums. The `Enumeration` base class was removed in 13.0.

```php
// old
class MyState extends \TYPO3\CMS\Core\Type\Enumeration {
    const __default = self::ACTIVE;
    const ACTIVE = 'active';
}

// new
enum MyState: string {
    case Active = 'active';
}
```

## Icons

Register icons in `Configuration/Icons.php`, not in `ext_localconf.php`:

```php
// Configuration/Icons.php
return [
    'my-icon' => [
        'provider' => \TYPO3\CMS\Core\Imaging\IconProvider\SvgIconProvider::class,
        'source' => 'EXT:my_ext/Resources/Public/Icons/my-icon.svg',
    ],
];
```

---

## Page TSconfig and User TSconfig

Do **not** use `ExtensionManagementUtility::addPageTSConfig()` or `addUserTSConfig()` in `ext_localconf.php`. Place files in:

- `Configuration/page.tsconfig` (loaded automatically for the whole installation)
- `Configuration/user.tsconfig`

Or ship them as part of a **Site Set** for per-site configuration.

---

## Site Sets (13.1+)

Structure reusable TypoScript, constants, and settings as a Site Set:

```
EXT:my_ext/Configuration/Sets/MyFeature/
    config.yaml         # set metadata, dependencies
    setup.typoscript
    constants.typoscript
    settings.yaml       # typed setting definitions with defaults
```

`config.yaml`:
```yaml
name: my-vendor/my-feature
label: My Feature
dependencies:
  - typo3/fluid-styled-content
```

Site sets replace the old `static_template` / `includeStaticInCLO` pattern.

## Database / Doctrine DBAL 4

- Use `QueryBuilder` from `ConnectionPool` — never raw SQL.
- `ExpressionBuilder` new helpers: `if()`, `castText()`, UNION support.
- `CHAR`/`BINARY` columns in `ext_tables.sql` are now created as fixed-length. Verify your SQL files if you relied on them being treated as VARCHAR.
- `tableoptions` / `collate` connection config keys are deprecated → use `defaultTableOptions` / `collation`.
- Use Doctrine DBAL `EnumType` (`\Doctrine\DBAL\Types\EnumType`) instead of the TYPO3 custom `\TYPO3\CMS\Core\Database\Schema\Types\EnumType` (deprecated in 13.4).

---

## Extbase File Uploads (13.3+)

Use the new built-in Extbase file upload API instead of custom solutions:

```php
use TYPO3\CMS\Extbase\Annotation\FileUpload;
use TYPO3\CMS\Extbase\Domain\Model\FileReference;

class MyModel extends AbstractEntity
{
    #[FileUpload([
        'validation' => ['required' => true, 'maxFiles' => 1, 'allowedMimeTypes' => ['image/*']],
        'uploadFolder' => '1:/user_uploads/',
    ])]
    protected ?FileReference $image = null;
}
```

---

## JavaScript / Assets

- **RequireJS** is removed. Use ES modules with `@typo3/` package imports.
- **jQuery** is removed from the frontend. Write vanilla JS or import a bundled copy if needed.
- Use `AssetCollector` (`addStyleSheet()`, `addJavaScript()`) for adding assets. Pass `external: true` to skip cache-busting on CDN assets.
- `<f:asset.css inline="true">` / `<f:asset.script inline="true">` inline a file's content directly.

---

## Logging

Use `RotatingFileWriter` to prevent unbounded log growth:

```php
// additional.php
$GLOBALS['TYPO3_CONF_VARS']['LOG']['TYPO3']['CMS']['writerConfiguration'][\Psr\Log\LogLevel::WARNING] = [
    \TYPO3\CMS\Core\Log\Writer\RotatingFileWriter::class => [
        'logFileInfix' => 'my_ext',
        'maxFiles' => 5,
    ],
];
```

---

## Content Security Policy

When adding inline scripts/styles that need a nonce, use the CSP nonce from the request — do not hard-code or disable CSP. From 13.4.x, nonce sources are only emitted when actually consumed to preserve page-caching performance.

---

## Symfony Messenger / Queues

If using message queues with rate limiting, configure rate limiters directly in the `messenger.yaml` / `Services.yaml` configuration rather than relying on hooks (which no longer exist).

---

## Quick Reference: Deprecated → Replacement

| Deprecated | Replacement |
|---|---|
| `StandaloneView` | `ViewFactoryInterface` |
| `$GLOBALS['TSFE']` | `$request->getAttribute('frontend.page.information')` |
| `addCacheTags()` on TSFE | `$request->getAttribute('frontend.cache.collector')` |
| `INCLUDE_TYPOSCRIPT` | `@import` |
| `list` plugin / `list_type` | dedicated `CType` per plugin |
| TCA sub types for plugins | dedicated `CType` per plugin |
| `Enumeration` base class | PHP 8.1 backed `enum` |
| `IconRegistry` in `ext_localconf.php` | `Configuration/Icons.php` |
| `addPageTSConfig()` in `ext_localconf.php` | `Configuration/page.tsconfig` |
| `addUserTSConfig()` in `ext_localconf.php` | `Configuration/user.tsconfig` |
| `GeneralUtility::hmac()` | `\TYPO3\CMS\Core\Crypto\HashService::hmac()` |
| `RenderingContext->getRequest()` | inject `ServerRequestInterface` directly |
| `renderStatic()` in ViewHelpers | `render()` |
| Hooks (`$GLOBALS['TYPO3_CONF_VARS']['SC_OPTIONS']`) | PSR-14 events with `#[AsEventListener]` |
| `static_template` / `includeStaticInCLO` | Site Sets |
| Manual system TCA columns | Auto-created from `ctrl` settings |
| `RequireJS` / AMD modules | ES modules with `@typo3/` imports |
